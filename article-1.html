Membangun Portofolio Web Statis dengan Custom Domain

Tutorial langkah demi langkah: Dari nol hingga online menggunakan GitHub Pages, Cloudflare, dan domain murah (berekstensi .my.id / .biz.id).

Mengapa Memilih Metode Ini?

Sebagai developer atau profesional, memiliki website portofolio adalah kewajiban. Namun, biaya hosting bulanan bisa memberatkan, terutama untuk pemula. Solusinya adalah Static Site. Dengan kombinasi GitHub Pages (hosting gratis) dan Cloudflare (manajemen DNS & keamanan (WAF - Web Application Firewall)), untuk personal branding Anda hanya perlu membayar biaya perpanjangan domain setahun sekali.

Tip Hemat: Gunakan ekstensi domain lokal seperti .my.id atau .biz.id yang harganya sangat terjangkau (sekitar Rp 3.000 s/d Rp 20.000 per tahun) dari registrar/provider di Indonesia.
1

Persiapan File Website

Anda membutuhkan file HTML sederhana. Jika belum punya, buatlah folder baru di komputer Anda bernama portofolio/blog/article , lalu buat file bernama index.html di dalamnya.

index.html 
<!DOCTYPE html>
<html>
<head>
    <title>Portofolio Saya</title>
    <style>
        body { font-family: sans-serif; text-align: center; padding: 50px; }
    </style>
</head>
<body>
    <h1>Halo, Saya Budi!</h1>
    <p>Selamat datang di portofolio web statis saya.</p>
</body>
</html>
2

Buat Repository di GitHub

Login ke akun GitHub Anda.

  • Klik tombol + di pojok kanan atas, pilih New repository.
  • Beri nama repository, misal: portofolio-saya.
  • Pilih Public.
  • Centang Add a README file (opsional tapi disarankan).
  • Klik Create repository.
[Tampilan Halaman New Repository GitHub]
User mengisi nama repo dan memilih 'Public'

Setelah repo jadi, klik Add file > Upload files. Tarik file index.html yang Anda buat tadi ke area upload, lalu klik tombol hijau Commit changes.

3

Aktifkan GitHub Pages

Agar file HTML bisa diakses sebagai website:

  1. Buka tab Settings di repository Anda.
  2. Di menu sidebar kiri, cari bagian "Code and automation", pilih Pages.
  3. Pada bagian Build and deployment, di bawah Branch, ubah dari None menjadi main (atau master).
  4. Klik Save.

Tunggu sekitar 1-2 menit, cek proses deploymentnya jika perlu (progress CI/CD bawaan github). Kemudian refresh halaman, Anda akan melihat link seperti ini:
https://username.github.io/portofolio-saya/

4

Beli Domain (.my.id / .biz.id)

Beli domain di penyedia lokal (seperti IDWebhost, JetOrbit, Niagahoster, IDCloudHost, DomaiNesia, JagoanHosting, dll).

Mengapa ekstensi ini?

Domain .my.id ditujukan untuk personal/blog, harganya seringkali di bawah Rp 15.000/tahun jika masih promo (karena termasuk ekstensi yang baru launching). .biz.id untuk bisnis UKM, juga sangat murah. Syarat aktivasinya pun mudah (biasanya hanya verifikasi email/KTP tergantung provider).

PENTING: Jangan atur DNS record dulu di panel penyedia domain. Kita akan mengaturnya di Cloudflare. Karena kita akan gunakan Cloudflare sebagai DNS Management, Proteksi DDoS, WAF dan proteksi lainnya.

5

Integrasi Cloudflare

Cloudflare bertindak sebagai "satpam" dan pengelola lalu lintas data agar website cepat dan aman (dapat SSL gratis).

  1. Buat akun di Cloudflare.com (Gratis).
  2. Klik Add a Site dan masukkan nama domain yang baru dibeli (misal: budidev.my.id).
  3. Pilih paket Free (paling bawah), lalu klik Continue.
  4. Cloudflare akan memindai DNS. Klik Continue saja sampai di halaman instruksi Change your nameservers.

Nameserver 1: elsa.ns.cloudflare.com

Nameserver 2: walt.ns.cloudflare.com

(Contoh nama server, milik Anda mungkin berbeda)

Salin kedua Nameserver dari Cloudflare tersebut. Login kembali ke panel tempat Anda membeli domain (registrar/provider), cari menu Nameserver Management, dan ganti nameserver bawaan dengan milik Cloudflare. Simpan. Tunggu waktu propagasi selesai, biasanya akan ada notifikasi via e-mail dari Cloudflare kalau domain sudah go-live.

6

Setting DNS Record

Kembali ke dashboard Cloudflare, masuk ke menu DNS > Records. Hapus semua record lama jika ada, lalu tambahkan record CNAME baru berikut untuk menghubungkan domain ke GitHub Pages (lihat entry paling bawah):

Type Name Content (IP GitHub) Proxy Status
A @ 185.199.108.153 Proxied
A @ 185.199.109.153 Proxied
A @ 185.199.110.153 Proxied
A @ 185.199.111.153 Proxied
CNAME www username.github.io Proxied

*Ganti username.github.io dengan username GitHub asli Anda. Name www ini sebagai sub-domain Anda, akan aktif dengan https://www.domain.my.id

Tentang SSL/TLS: Di menu Cloudflare sebelah kiri, klik SSL/TLS. Jika Anda mengalami error "Too many redirects", untuk menghindarinya pastikan mode enkripsi diatur ke Full (bukan Flexible). Jika tidak ada kendala redirects, cukup set ke Flexible.
7

Finalisasi di GitHub

Langkah terakhir untuk memberi tahu GitHub bahwa Anda menggunakan domain tersebut.

  1. Kembali ke Repository GitHub > Settings > Pages.
  2. Scroll ke bawah ke bagian Custom domain.
  3. Masukkan nama domain Anda (misal: budidev.my.id atau www.budidev.my.id).
  4. Klik Save. GitHub akan melakukan pengecekan DNS (bisa memakan waktu 1-15 menit).
  5. Setelah centang hijau muncul, centang kotak Enforce HTTPS di bawahnya.
